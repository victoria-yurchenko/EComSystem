1. Главная страница (/)

Юзкейсы:

    Просмотр информации о системе:
        Акторы: Гость/Зарегистрированный пользователь.
        Цель: Ознакомиться с описанием платформы и её функционалом.
        Шаги:
            Пользователь открывает главную страницу.
            Система отображает краткое описание платформы (секции "О нас", преимущества и отзывы).

2. Регистрация и вход (/auth)

Юзкейсы:

    Регистрация пользователя:
        Акторы: Новый пользователь.
        Цель: Создать учётную запись.
        Шаги:
            Пользователь вводит e-mail и пароль в форму регистрации.
            Нажимает "Регистрация".
            Система проверяет уникальность e-mail через API /api/auth/register.
            Успешная регистрация: система перенаправляет пользователя на страницу входа.

    Вход в систему:
        Акторы: Зарегистрированный пользователь.
        Цель: Авторизоваться для доступа к личному кабинету.
        Шаги:
            Пользователь вводит e-mail и пароль в форму входа.
            Нажимает "Войти".
            Система проверяет учетные данные через API /api/auth/login и выдает JWT.
            Успешный вход: перенаправление в личный кабинет /profile.

3. Страница подписок (/subscriptions)

Юзкейсы:

    Просмотр доступных подписок:
        Акторы: Зарегистрированный пользователь.
        Цель: Ознакомиться с тарифами.
        Шаги:
            Пользователь открывает страницу подписок.
            Система запрашивает данные о тарифах через API /api/subscriptions.
            Пользователь видит список подписок с описанием, стоимостью и продолжительностью.

    Оформление подписки:
        Акторы: Зарегистрированный пользователь.
        Цель: Выбрать подписку и оплатить.
        Шаги:
            Пользователь нажимает "Оформить" на выбранной подписке.
            Открывается форма оплаты (Stripe/PayPal).
            После успешной оплаты через API /api/subscriptions/subscribe создаётся подписка.
            Система показывает подтверждение.

4. Личный кабинет (/profile)

Юзкейсы:

    Просмотр активной подписки:
        Акторы: Зарегистрированный пользователь.
        Цель: Проверить статус текущей подписки.
        Шаги:
            Пользователь открывает личный кабинет.
            Система запрашивает активную подписку через API /api/subscriptions/active.
            Пользователь видит статус, дату начала и окончания подписки.

    Управление подпиской:
        Акторы: Зарегистрированный пользователь.
        Цель: Отключить автопродление или отменить подписку.
        Шаги:
            Пользователь нажимает "Управление подпиской".
            Через API /api/subscriptions/manage пользователь отключает автопродление или отменяет подписку.

5. История транзакций (/transactions)

Юзкейсы:

    Просмотр истории платежей:
        Акторы: Зарегистрированный пользователь.
        Цель: Ознакомиться с прошлыми транзакциями.
        Шаги:
            Пользователь открывает страницу транзакций.
            Система запрашивает данные через API /api/transactions.
            Пользователь видит список транзакций с фильтром по статусу (успешные, неудачные).

6. Уведомления (/notifications)

Юзкейсы:

    Просмотр уведомлений:
        Акторы: Зарегистрированный пользователь.
        Цель: Узнать об истекающих подписках или других важных событиях.
        Шаги:
            Пользователь открывает страницу уведомлений.
            Система запрашивает уведомления через API /api/notifications.
            Пользователь видит список уведомлений.

    Пометка уведомлений как прочитанных:
        Акторы: Зарегистрированный пользователь.
        Цель: Очистить список непрочитанных уведомлений.
        Шаги:
            Пользователь нажимает "Отметить как прочитанное".
            Через API /api/notifications/mark-as-read уведомление отмечается как прочитанное.

7. Админ-панель (/admin)

Юзкейсы:

    Управление подписками:
        Акторы: Администратор.
        Цель: Добавить, изменить или удалить тарифы.
        Шаги:
            Администратор открывает панель управления.
            Через API /api/subscriptions редактирует список подписок.

    Просмотр аналитики:
        Акторы: Администратор.
        Цель: Получить отчёты.
        Шаги:
            Администратор запрашивает отчёт через API /api/reports.
            Система отображает аналитику: общее количество пользователей, доходы, популярные тарифы.

8. Ошибки и сообщения

Юзкейсы:

    Обработка ошибок API:
        Акторы: Все пользователи.
        Цель: Отобразить сообщение при ошибке.
        Шаги:
            При ошибке API система отображает уведомление (например, через Snackbar Material-UI).
